# CCO-K 실제 동작 검증 체크리스트

이 문서는 문서가 아니라 실제 명령이 의도대로 작동하는지 확인하기 위한 점검표입니다.

## 1. 첫 진입 흐름
### 테스트 명령
- /시작하기
- /메뉴판
- /길잡이
- /추천

### 기대 결과
- /시작하기: 설치 → /ccok-setup → /메뉴판 순서를 보여줘야 함
- /메뉴판: 초급 시작 / 작업/마무리 / 고급 자동화 3단 구조로 보여야 함
- /길잡이: 완전 처음인 사람용 질문/추천 흐름이어야 함
- /추천: 이미 작업 중인 사람에게 다음 행동 1개를 추천해야 함

## 2. 문제 해결 흐름
### 테스트 명령
- /고쳐
- /원인추적 [문제]
- /딥분석 [문제]

### 기대 결과
- /고쳐: 빠른 수정 중심
- /원인추적: 왜 이런 일이 생겼는지 추적 중심
- /딥분석: 구조/성능/반복 장애 깊은 분석 중심
- 세 명령 차이가 설명에 드러나야 함

## 3. OMC 연결 흐름
### 테스트 명령
- /ccok-setup
- /자동개발 [내용]
- /끝까지 [내용]
- /초고속 [내용]
- /계획모드 [내용]

### 기대 결과
- OMC 미연결 상태면 먼저 /ccok-setup 기준으로 안내해야 함
- ./install.sh 만 던지고 끝나면 안 됨
- 끝나면 /메뉴판 또는 다음 명령으로 이어지는 흐름이 보여야 함

## 4. 별칭 정리 흐름
### 테스트 명령
- /메뉴
- /진단
- /끝장내기
- /팀플레이

### 기대 결과
- 각 명령은 대표 명령으로 유도해야 함
- 예:
  - /메뉴 -> /메뉴판
  - /진단 -> /추천
  - /끝장내기 -> /끝까지
  - /팀플레이 -> /팀실행

## 5. 최종 확인 포인트
- 문서와 실제 출력이 다르지 않은가
- 메뉴판의 명령 목록과 CLAUDE.md 라우팅이 같은가
- 초보자가 읽었을 때 "지금 뭘 치면 되는지" 바로 보이는가
- 명령어를 여러 개 나열하지 않고 다음 행동 1개를 제시하는가
